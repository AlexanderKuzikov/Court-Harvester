## ✨ Возможности

| Возможность | Описание |
|-------------|----------|
| 🔍 **Сбор с нуля** | Поиск всех судов РФ через DaData API с итеративным расширением префиксов |
| 🔄 **Инкрементальное обновление** | Проверка «хвостов» (MAX+200), новых территорий и пропусков |
| ✅ **Валидация существующих** | Перепроверка всех записей на изменения адресов, переименования, закрытия |
| 🔑 **Авто-ротация ключей** | До 4 API-ключей с автоматическим переключением при исчерпании лимита |
| ⏱ **Rate limiting** | Bottleneck + axios-retry для соблюдения ограничений DaData |
| 💾 **Чекпоинты** | Автосохранение прогресса каждые N запросов |
| 📊 **Экспорт** | JSON (отсортированный) + Excel (два листа: все суды + изменения) |
| 📝 **Логирование** | Полный лог с временными метками по каждому запуску |

## 🏗 Архитектура

```
┌────────────────────────────────────────────────────────────────┐
│                       CLI (commander)                          │
│              src/index.ts — точка входа                        │
└────────────────────────┬───────────────────────────────────────┘
                         │
         ┌───────────────┼───────────────┐
         ▼               ▼               ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────────┐
│  RegionHarvester │ │  FullHarvester  │ │  update.ts          │
│  (сбор региона)  │ │  (полный сбор)  │ │  (обновление базы)  │
└────────┬─────────┘ └────────┬────────┘ └──────────┬──────────┘
         │                    │                      │
         ▼                    ▼                      ▼
┌──────────────────────────────────────────────────────────┐
│                    ApiClient                              │
│  DaData HTTP · Rate limiter · Retry · Обработка ошибок    │
└────────────────────────┬─────────────────────────────────┘
                         │
         ┌───────────────┴───────────────┐
         ▼                               ▼
┌─────────────────┐             ┌─────────────────────┐
│  KeyRotation    │             │  DaData API          │
│  Manager        │◄────────────│  suggestions.dadata  │
│  (1→2→3→4.env) │    HTTP     │  .ru                 │
└─────────────────┘             └─────────────────────┘
```

### Компоненты

| Компонент | Файл | Назначение |
|-----------|------|------------|
| **ApiClient** | `src/core/ApiClient.ts` | HTTP-клиент с rate limiting (Bottleneck), retry (axios-retry), обработкой 403/quota |
| **KeyRotationManager** | `src/core/KeyRotationManager.ts` | Автоматическая ротация ключей из `keys/2–4.env` по 9 500 запросов |
| **RegionHarvester** | `src/core/RegionHarvester.ts` | Сбор судов по одному региону через поисковые запросы (города) |
| **FullHarvester** | `src/core/FullHarvester.ts` | Полный сбор всех судов РФ через алфавитный перебор префиксов |
| **update.ts** | `src/examples/update.ts` | Основной скрипт обновления (4 шага) |

### Процесс сбора (фазы)

```
Фаза 1–3: АА–ЯЯ + регионы 01–99  →  7 761 судов
Фаза 4:   Дополнительные запросы   →      +1
Фаза 5:   Пропуски MIN–MAX         →  +2 188  ← самая результативная
Фаза 6:   Хвосты выше MAX          →    +127
Фаза 7:   Ниже MIN                 →    +129
────────────────────────────────────────────────
Фаза 8:   Новые территории 90–99   →   уточнение
Фаза 9B:  Топ-20 регионов хвосты   →   уточнение
────────────────────────────────────────────────
Итого:                             10 206 судов
```

## 🚀 Быстрый старт

### Предварительные требования

- **Node.js** ≥ 18
- **npm**
- **Ключи DaData**: [получить на dadata.ru][dadata]

### Установка

```bash
git clone https://github.com/AlexanderKuzikov/Court-Harvester.git
cd Court-Harvester
npm install
```

### Настройка ключей

Создайте файлы в папке `keys/`:

```bash
keys/
├── 1.env          # Основной (не используется скриптами — резерв)
├── 2.env          # Рабочий ключ
├── 3.env          # Дополнительный
└── 4.env          # Дополнительный
```

Формат каждого `.env`:

```env
DADATA_API_KEY=ваш_api_ключ
DADATA_SECRET_KEY=ваш_secret_ключ
```

> **⚠️ Важно:** Скрипты обновления используют `keys/2.env`, `3.env`, `4.env`.
> `keys/1.env` — резервный, скрипты его не трогают.

## 🔄 Обновление базы

### 🔍 Поиск новых судов (ежемесячно)

```bash
npm run update
```

**~8 684 запросов · 1 ключ · ~18 минут**

| Шаг | Действие | Описание |
|:---:|----------|----------|
| 1 | **Хвосты** | Для каждого префикса проверяет MAX+1 … MAX+200. Стоп после 20 пустых подряд. |
| 2 | **Территории** | Проверяет регионы 90–99 на появление новых субъектов РФ. |
| 3 | **Дырки** | Для каждого префикса проверяет все номера от 0000 до MAX. |

```
Есть: 01RS0001 … 01RS0045
Хвост: 01RS0046 … 01RS0245  (MAX+200)
Дырки: 01RS0000 … 01RS0045  (пропущенные)
```

### 📋 Полное обновление (ежеквартально)

```bash
npm run update:full
```

**~18 890 запросов · 2–3 ключа · ~36 минут**

Всё из `npm run update` плюс:

| Шаг | Действие | Результат |
|:---:|----------|-----------|
| 4 | **Ревалидация** | Каждый из 10 206 судов проверяется на актуальность |

| Ситуация | Статус |
|----------|--------|
| Данные не изменились | `existing` |
| Данные изменились | `updated` |
| Суд не найден в API | `not_found` (возможно закрыт) |

### 📊 Анализ перед обновлением

```bash
npm run analyze-prefixes
```

Показывает:
- Распределение MAX по префиксам
- Оценку количества запросов для каждого шага
- Укладывается ли план в дневной лимит DaData (10 000 запросов)

## 🔑 Ключи API и ротация

**KeyRotationManager** автоматически управляет ключами:

```
keys/2.env          keys/3.env          keys/4.env
  0 – 9 500    →   9 501 – 19 000  →   19 001 – 28 500
```

- **Лимит на ключ:** 9 500 запросов (запас 500 от дневного лимита 10 000)
- **Автопереключение:** при исчерпании — без остановки процесса
- **Исчерпание всех ключей:** корректное завершение с сохранением прогресса

## 🔤 Структура кода судов

```
01 RS 0001
│  │   │
│  │   └── Порядковый номер (0000–9999)
│  └────── Тип суда (RS, MS, AS, …)
└───────── Код региона (01–99)
```

| Код | Суд |

| `01RS0001` | Первый районный суд Республики Адыгея |
| `77MS0042` | 42-й мировой судья Москвы |
| `78AS0001` | Арбитражный суд Санкт-Петербурга |

## 🧪 Примеры использования

```bash
# Запуск CLI
npm run start

# Сбор по одному региону
npm run "harvest:59"

# Полный сбор (разные фазы)
npm run "harvest:full:phase4"
npm run "harvest:full:phase8"
npm run "harvest:full:phase9b"

# Анализ и диагностика
npm run analyze-prefixes   # Оценка запросов
npm run find-gaps          # Поиск пропусков в кодах
npm run check-ai           # Проверка судов по ИНН
npm run sort-export        # Сортировка + Excel-экспорт
npm run count-prefixes     # Статистика префиксов

# Проверка одного суда
npx tsx src/examples/test-single-court.ts
```
